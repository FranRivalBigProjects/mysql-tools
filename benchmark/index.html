How the Simulation Works

The benchmark model applies empirically-derived multipliers for each workload type, query pattern, and hardware configuration to estimate realistic MySQL throughput.

  • Base QPS derived from InnoDB single-thread performance at the selected storage tier
  • Index miss penalty: each unindexed query scans up to N rows vs a single B-tree lookup
  • Buffer pool hit ratio: data fitting in RAM vs requiring disk I/O
  • Connection overhead: thread scheduler contention above max_connections threshold
  • Write amplification: InnoDB redo log and doublewrite buffer cost for INSERT/UPDATE
