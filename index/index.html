Index Optimization Rules

These are the principles the analyzer applies to find missing and redundant indexes.

  • WHERE equality columns should be the first columns in a composite index
  • Range columns (BETWEEN, >, <, LIKE) go after equality columns in the index
  • ORDER BY columns can be included in the index after WHERE columns to avoid filesort
  • A covering index includes all columns in SELECT to avoid row fetches (Using index)
  • An index on (a) is made redundant by an index on (a, b) for equality lookups
  • JOIN keys need indexes on the foreign key column of the joined table
  • Low-cardinality columns (status, boolean flags) are weak leading index columns
